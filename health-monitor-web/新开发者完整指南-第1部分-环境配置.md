## 🎯 系统概述

### 系统架构

```
手表端 (ZeppOS)          手机端 (Android/iOS)      Web服务器 (Node.js)        Web前端 (Vue3)
     │                         │                          │                          │
     │  采集传感器数据           │                          │                          │
     │  (每5分钟自动)            │                          │                          │
     │                         │                          │                          │
     │──────BLE推送───────>     │                          │                          │
     │                         │                          │                          │
     │                         │  HTTP POST               │                          │
     │                         │──────────────────>       │                          │
     │                         │  /api/health-data        │                          │
     │                         │                          │                          │
     │                         │                          │  保存到SQLite数据库       │
     │                         │                          │  进行安全检测             │
     │                         │                          │  触发报警（如需要）       │
     │                         │                          │                          │
     │                         │                          │<─────WebSocket──────────│
     │                         │                          │  实时推送数据更新         │
     │                         │                          │                          │
```

### 技术栈

**手表端**:
- ZeppOS 3.0+
- JavaScript (ZML Framework)
- 传感器: 心率、血氧、压力、步数、气压等

**手机端**:
- Zepp App (Side Service)
- BLE 蓝牙通信
- HTTP 网络请求

**Web后端**:
- Node.js 16+
- SQLite3 数据库
- WebSocket 实时通信
- DeepSeek AI API

**Web前端**:
- Vue 3 (Composition API)
- Element Plus UI
- ECharts 图表
- Pinia 状态管理
- Vite 构建工具

## 🔧 核心配置调整（新电脑必改）

### 1. 手表端服务器地址配置 ⚠️ **必须修改**

**文件**: `calories/app-side/index.js`

```javascript
// 第 3 行
const SERVER_URL = "http://192.168.31.206:3000/api/health-data";
```

**如何修改**:

1. **获取新电脑的IP地址**:
   ```powershell
   ipconfig
   # 找到 "IPv4 地址"，例如: 192.168.1.100
   ```

2. **修改配置**:
   ```javascript
   // 改为你的新IP地址
   const SERVER_URL = "http://192.168.1.100:3000/api/health-data";
   ```

3. **重新编译手表应用**:

## 🐛 常见问题排查

### Q1: 端口被占用

**错误**: `Error: listen EADDRINUSE: address already in use 0.0.0.0:3000`

**解决**:
```powershell
# 查找占用3000端口的进程
netstat -ano | findstr :3000

# 结束进程
taskkill /PID <PID> /F

# 或修改端口配置（见上文"端口配置"）
```

### Q2: 数据库初始化失败

**错误**: `SQLITE_ERROR` 或 `table already exists`

**解决**:
```powershell
# 删除旧数据库重新初始化
Remove-Item server\health_monitor.db
npm run init-db
```

### Q3: 手表数据推送失败

**检查清单**:
- [ ] 手表和手机蓝牙已连接？
- [ ] 手机有网络（WiFi或4G）？
- [ ] 后端服务器在运行？
- [ ] 手表端配置的IP地址正确？
- [ ] 手机和电脑在同一WiFi网络？

**测试连接**:
```powershell
# 在手机浏览器访问
http://<你的电脑IP>:3000/ping
# 应该返回"服务器运行正常"
```

### Q4: 前端无法连接后端

**错误**: 浏览器控制台显示 `Failed to fetch` 或 `CORS error`

**解决**:
1. 确认后端服务器在运行
2. 检查 `frontend/vite.config.js` 的 proxy 配置
3. 清除浏览器缓存后刷新
