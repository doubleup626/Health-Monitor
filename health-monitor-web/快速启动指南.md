## ⚙️ **如何停止服务器**

### **方法1：使用任务管理器**

```
1. 打开任务管理器 (Ctrl+Shift+Esc)
2. 找到 Node.js 进程
3. 结束任务
```

### **方法2：使用 PowerShell**

```powershell
# 查找并结束端口 3000 的进程（后端）
netstat -ano | findstr :3000
# 记住 PID，然后：
taskkill /PID <PID> /F

# 查找并结束端口 8080 的进程（前端）
netstat -ano | findstr :8080
# 记住 PID，然后：
taskkill /PID <PID> /F
```

## 🔄 **如何重启服务器**

```powershell
# 1. 停止当前服务器（使用上面的方法）

# 2. 重新启动
cd d:/Zepp_workspace/health-monitor-web

# 启动后端（后台）
cd server
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd '$PWD'; node server.js"

# 启动前端（新窗口）
cd ../frontend
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd '$PWD'; npm run dev"
```

## ❌ **常见问题**

### **Q1: 端口被占用**

```
错误: EADDRINUSE: address already in use :::3000

解决:
1. 找到占用端口的进程
   netstat -ano | findstr :3000
2. 结束该进程
   taskkill /PID <PID> /F
3. 重新启动服务器
```

### **Q2: 前端无法连接后端**

```
错误: Failed to fetch / Network Error

解决:
1. 确认后端服务器正在运行
   netstat -ano | findstr :3000
2. 检查后端是否正常启动（查看终端输出）
3. 检查防火墙设置
```

### **Q3: 数据库锁定**

```
错误: SQLITE_BUSY: database is locked

解决:
1. 关闭所有打开数据库的工具（DB Browser）
2. 重启后端服务器
```
