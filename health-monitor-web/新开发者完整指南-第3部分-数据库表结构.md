## 📊 数据库概述

### 数据库类型

- **数据库引擎**: SQLite 3
- **数据库文件**: `health_monitor.db`
- **文件位置**: `D:\Zepp_workspace\health-monitor-web\server\health_monitor.db`
- **初始化脚本**: `database-init.sql`
- **操作模块**: `database.js`

### 为什么选择 SQLite

1. **零配置**: 无需安装单独的数据库服务器
2. **轻量级**: 单文件数据库，易于备份和迁移
3. **高性能**: 对于中小规模数据（<100万条）性能优秀
4. **跨平台**: 可在 Windows/Linux/macOS 上运行
5. **嵌入式**: 完美适合本地部署和开发测试

### 表结构总览

| 序号 | 表名 | 用途 | 记录数量级 |
|------|------|------|-----------|
| 1 | workers | 工人基本信息 | 数十~数百 |
| 2 | devices | 设备信息 | 数十~数百 |
| 3 | health_realtime | 实时健康数据 | 与工人数量相同 |
| 4 | health_history | 历史健康数据 | 数千~数百万 |
| 5 | alerts | 报警记录 | 数百~数万 |
| 6 | ai_analysis | AI分析记录 | 数百~数千 |

## 📋 表1: workers（工人信息表）

```sql
CREATE TABLE IF NOT EXISTS workers (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  worker_id VARCHAR(50) UNIQUE NOT NULL,      -- 工人唯一ID
```

## 📱 表2: devices（设备信息表）

### 表结构

```sql
CREATE TABLE IF NOT EXISTS devices (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  device_id VARCHAR(50) UNIQUE NOT NULL,       -- 设备ID
```

## 💓 表3: health_realtime（实时健康数据表）

### 表结构

```sql
CREATE TABLE IF NOT EXISTS health_realtime (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  worker_id VARCHAR(50) NOT NULL,
  device_id VARCHAR(50) NOT NULL,
  
  -- 生理数据
  heart_rate_current INTEGER,            -- 当前心率
  heart_rate_last INTEGER,               -- 最近心率
  heart_rate_resting INTEGER,            -- 静息心率
  heart_rate_daily_max INTEGER,          -- 今日最高心率
  blood_oxygen_value INTEGER,            -- 血氧值
  blood_oxygen_time BIGINT,              -- 血氧测量时间
  stress_value INTEGER,                  -- 压力值
  stress_time BIGINT,                    -- 压力测量时间
  stress_today_avg INTEGER,              -- 今日平均压力
  body_temperature REAL,                 -- 体温
  
  -- 活动数据
  steps INTEGER,                         -- 步数
  calories INTEGER,                      -- 卡路里
  distance REAL,                         -- 距离
  
  -- 环境数据
  air_pressure INTEGER,                  -- 气压
  altitude INTEGER,                      -- 海拔（负值=深度）
  
  -- 状态标记
  is_safe BOOLEAN DEFAULT 1,             -- 是否安全
  alert_level INTEGER DEFAULT 0,         -- 报警级别
  alert_message TEXT,                    -- 报警信息
  
  -- 时间戳
  timestamp BIGINT NOT NULL,             -- 数据时间戳（毫秒）
  received_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  
  FOREIGN KEY (worker_id) REFERENCES workers(worker_id),
  FOREIGN KEY (device_id) REFERENCES devices(device_id)
);
```

### 字段说明

#### 生理数据字段

| 字段名 | 类型 | 单位 | 说明 | 正常范围 |
|--------|------|------|------|----------|
| heart_rate_current | INTEGER | bpm | 当前心率（可能为0） | 60-100 |
| heart_rate_last | INTEGER | bpm | 最近一次心率 ⭐ | 60-100 |
| heart_rate_resting | INTEGER | bpm | 静息心率 | 50-70 |
| heart_rate_daily_max | INTEGER | bpm | 今日最高心率 | - |
| blood_oxygen_value | INTEGER | % | 血氧饱和度 | >95% |
| blood_oxygen_time | BIGINT | 秒 | 血氧测量时间戳 | - |
| stress_value | INTEGER | - | 压力指数 | <80 |
| stress_time | BIGINT | 秒 | 压力测量时间戳 | - |
| stress_today_avg | INTEGER | - | 今日平均压力 | - |
| body_temperature | REAL | °C | 体温 | 36.0-37.5 |

#### 活动数据字段

| 字段名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| steps | INTEGER | 步 | 当前步数（实时） |
| calories | INTEGER | kcal | 消耗卡路里（实时） |
| distance | REAL | 米 | 移动距离（实时） |

#### 环境数据字段

| 字段名 | 类型 | 单位 | 说明 |
|--------|------|------|------|
| air_pressure | INTEGER | hPa | 气压（实时） |
| altitude | INTEGER | 米 | 海拔（负值表示地下） |

#### 安全状态字段

| 字段名 | 类型 | 说明 | 可能值 |
|--------|------|------|--------|
| is_safe | BOOLEAN | 是否安全 | 1=安全, 0=异常 |
| alert_level | INTEGER | 报警级别 | 0=无, 1=低, 2=中, 3=高 |
| alert_message | TEXT | 报警信息 | "心率过高：145 bpm" |

### 数据特点

**每个工人只有一条记录**（通过 `UPSERT` 操作更新）

- 优点：查询速度快，数据量小
- 更新策略：收到新数据时，更新该工人的记录
- 历史数据：保存在 `health_history` 表

### 数据库操作函数

```javascript
// 获取所有工人的实时数据
const realtimeData = await realtimeDB.getAll();

// 获取单个工人的实时数据
const workerData = await realtimeDB.getByWorkerId('W001');

// 更新或插入实时数据（UPSERT）
await realtimeDB.upsert({
  worker_id: 'W001',
  device_id: '10092800',
  heart_rate_current: 88,
  heart_rate_last: 88,
  heart_rate_resting: 55,
  blood_oxygen_value: 95,
  stress_value: 32,
  steps: 300,
  altitude: -100,
  is_safe: 1,
  alert_level: 0,
  timestamp: Date.now()
});
```

## 📚 表4: health_history（历史健康数据表）

### 表结构

```sql
CREATE TABLE IF NOT EXISTS health_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  worker_id VARCHAR(50) NOT NULL,
  device_id VARCHAR(50) NOT NULL,
  
  -- 完整的健康数据（JSON格式）
  data_json TEXT NOT NULL,
  
  -- 关键指标（便于查询）
  heart_rate INTEGER,
  blood_oxygen INTEGER,
  stress INTEGER,
  
  -- 时间戳
  timestamp BIGINT NOT NULL,
  received_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  
  FOREIGN KEY (worker_id) REFERENCES workers(worker_id)
);

-- 创建索引（加速查询）
CREATE INDEX IF NOT EXISTS idx_history_worker_time 
ON health_history(worker_id, timestamp);
```

### 字段说明

| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| id | INTEGER | 自增主键 | 1 |
| worker_id | VARCHAR(50) | 工人ID | "W001" |
| device_id | VARCHAR(50) | 设备ID | "10092800" |
| data_json | TEXT | 完整的原始数据（JSON） | "{...}" |

| blood_oxygen | INTEGER | 血氧（便于快速查询） | 95 |
| stress | INTEGER | 压力（便于快速查询） | 32 |

## 🤖 表6: ai_analysis（AI分析记录表）

```sql
CREATE TABLE IF NOT EXISTS ai_analysis (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  worker_id VARCHAR(50) NOT NULL,
  
  -- 分析请求
  request_data TEXT NOT NULL,            -- 请求时的实时数据
  prompt TEXT NOT NULL,                  -- 发送给AI的提示词
  history_data TEXT,                     -- 历史数据
  
  -- 分析结果

  response_time INTEGER,                 -- 响应时间（毫秒）
  
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,

);
```

### 数据库性能优化

#### 1. 索引优化

```sql
-- 已创建的索引
CREATE INDEX idx_history_worker_time ON health_history(worker_id, timestamp);

-- 可选的额外索引
CREATE INDEX idx_alerts_worker ON alerts(worker_id);
CREATE INDEX idx_alerts_level ON alerts(alert_level);
```

#### 2. 清理历史数据

```javascript
// 定期清理（建议每周执行）
// 保留最近30天数据

const { run } = require('./database');

const cleanOldData = async () => {
  const thirtyDaysAgo = Date.now() - 30 * 24 * 60 * 60 * 1000;
  
  // 清理历史数据
  await run('DELETE FROM health_history WHERE timestamp < ?', [thirtyDaysAgo]);
  
  // 清理已处理的报警
  const sevenDaysAgo = Date.now() - 7 * 24 * 60 * 60 * 1000;
  await run(
    'DELETE FROM alerts WHERE is_handled = 1 AND created_at < ?',
    [new Date(sevenDaysAgo).toISOString()]
  );
  
  console.log('✅ 数据清理完成');
};
```

## 📊 数据统计查询示例

### 工人统计

```sql
-- 工人总数
SELECT COUNT(*) as total FROM workers;

-- 按班组统计
SELECT team, COUNT(*) as count 
FROM workers 
GROUP BY team;

-- 按岗位统计
SELECT position, COUNT(*) as count 
FROM workers 
GROUP BY position;
```

### 设备统计

```sql
-- 在线设备数（最近10分钟有数据）
SELECT COUNT(*) as online 
FROM devices 
WHERE is_active = 1 
AND last_seen > datetime('now', '-10 minutes');

-- 离线设备
SELECT COUNT(*) as offline 
FROM devices 
WHERE is_active = 0 
OR last_seen <= datetime('now', '-10 minutes');
```

### 报警统计

```sql
-- 按级别统计报警
SELECT alert_level, COUNT(*) as count 
FROM alerts 
GROUP BY alert_level;

-- 未处理的危险报警
SELECT COUNT(*) as danger_count 
FROM alerts 
WHERE alert_level = 3 
AND is_handled = 0;

-- 最近24小时报警趋势
SELECT 
  strftime('%H', created_at) as hour,
  COUNT(*) as count 
FROM alerts 
WHERE created_at > datetime('now', '-1 day')
GROUP BY hour;
```

### 历史数据统计

```sql
-- 某工人最近10条数据的平均心率
SELECT AVG(heart_rate) as avg_hr 
FROM health_history 
WHERE worker_id = 'W001' 
ORDER BY timestamp DESC 
LIMIT 10;

-- 今日数据总量
SELECT COUNT(*) as count 
FROM health_history 
WHERE date(received_at) = date('now');
```
