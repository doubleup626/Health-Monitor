## 📊 完整数据流程（7个阶段）

### 阶段1: 手表端数据采集

**触发时机**:
- 自动触发：每5分钟（分钟数是5的倍数）
- 手动触发：点击"手动采集一次"按钮
- 启动触发：点击"开始采集"后立即执行一次

**执行文件**: `calories/app-service/data-collector.js`

**详细流程**:

```javascript
// 步骤1: 定时器检查（每分钟触发）
timeSensor.onPerMinute(() => {
  const now = new Date()
  const minute = now.getMinutes()
  
  // 步骤2: 判断是否是采集时刻
  if (minute % 5 === 0) {
    console.log(`✅ 触发自动采集 - ${now.toLocaleTimeString()}`)
    collectAndSend()
  }

// 步骤3: 采集所有传感器数据
function collectAllData() {
  // 3.1 读取设备ID
  const deviceId = hmSetting.getDeviceInfo().deviceId
  
  // 3.2 读取心率传感器
  const heartRate = new HeartRate()
  const hrData = {
    current: heartRate.getCurrent(),      // 当前心率（可能为0）
    last: heartRate.getLast(),            // 最近心率 ⭐ 主要使用
    resting: heartRate.getResting(),      // 静息心率
    dailyMaximum: heartRate.getDailyMaximum()  // 今日最高
  }
  
  // 3.3 读取血氧传感器
  const bloodOxygen = new BloodOxygen()
  const spo2Data = {
    value: bloodOxygen.getCurrent(),      // ⚠️ 缓存值，需提前手动测量
    time: bloodOxygen.getLastTime(),      // 测量时间戳
    retCode: bloodOxygen.getRetCode(),    // 2=测量成功
    lastFewHour: bloodOxygen.getLastFewHour()  // 最近几小时数据
  }
  
  // 3.4 读取压力传感器
  const stress = new Stress()
  const stressData = {
    value: stress.getCurrent(),           // ⚠️ 缓存值
    time: stress.getTime(),
    todayAverage: stress.getTodayAverage(),
    lastWeekAverage: stress.getLastWeekAverage()
  }
  
  // 3.5 读取活动传感器（实时）
  const step = new Step()
  const calorie = new Calorie()
  const distance = new Distance()
  
  // 3.6 读取环境传感器（实时）
  const barometer = new Barometer()
  const barometerData = {
    airPressure: barometer.getPressure(),  // 气压（hPa）
    altitude: barometer.getAltitude()      // 海拔（负值=深度）
  }
  
  // 3.7 读取体温传感器（可能不支持）
  const bodyTemp = new BodyTemperature()
  const tempData = {
    current: bodyTemp.getCurrent(),        // Active2 通常为0
    timeinterval: bodyTemp.getTimeInterval()
  }
  
  // 3.8 组合完整数据
  return {
    timestamp: Date.now(),
    deviceId: deviceId,
    heartRate: hrData,
    bloodOxygen: spo2Data,
    stress: stressData,
    bodyTemperature: tempData,
    step: { current: step.getCurrent(), target: step.getTarget() },
    calorie: { current: calorie.getCurrent() },
    distance: distance.getCurrent(),
    barometer: barometerData,
    sleep: null  // 工作时段不采集睡眠
  }
}
```

### 阶段2: 手表端 → 手机端（BLE传输）

**执行文件**: `calories/app-service/data-collector.js`

**传输方式**: 蓝牙低功耗（BLE）

**详细流程**:

```javascript
// 步骤1: 准备数据
const healthData = collectAllData()

// 步骤2: 通过BLE发送到手机端
this.request({
  method: 'data.send',              // 自定义方法名
  params: {
    dataList: [healthData]          // 数据数组（支持批量）
  }
})
.then(response => {
  console.log('✅ 数据已发送到手机端')
})
.catch(error => {
  console.error('❌ 发送失败:', error)
  // 失败处理：保存到本地，稍后重试
})
```

**BLE通信特点**:
- 低功耗：适合长期运行
- 距离限制：通常 <10米
- 需要配对：手表和手机必须通过蓝牙连接
- 自动重连：连接断开后会自动重试

**详细流程**:

```javascript
// 步骤1: 接收手表端的BLE请求
AppSideService.onRequest(req, res) {
  console.log('📱 手机端收到手表数据')
  
  const params = req.body.params
  
  // 步骤2: 提取数据
  const dataList = params.dataList
  
  // 步骤3: 构建服务器请求格式
  const requestData = {
    type: 'batch',                    // 固定值
    count: dataList.length,           // 数据条数
    worker_id: 'W001',                // 可选：工人ID
    data: dataList                    // 完整数据数组
  }
  
  // 步骤4: 发送HTTP POST请求到服务器
  this.request({
    method: 'POST',
    url: 'http://192.168.31.206:3000/api/health-data',  // ⚠️ 新电脑需修改IP
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(requestData)
  })
  .then(response => {
    console.log('✅ 数据已推送到服务器')
    
    // 步骤5: 返回成功给手表端
    res({
      result: 'success',
      timestamp: Date.now()
    })
  })
  .catch(error => {
    console.error('❌ 推送失败:', error)
    
    // 返回失败给手表端
    res({
      result: 'failed',
      error: error.message
    })
  })
}
```

**网络要求**:
- 手机必须有网络（WiFi或4G/5G）
- 服务器必须在运行
- IP地址必须正确
- 防火墙允许3000端口

### 阶段5: 安全检测逻辑

**执行文件**: `health-monitor-web/server/safety-check.js`

**检测维度**: 5个

**详细逻辑**:

```javascript
function checkSafety(data) {
  const alerts = []
  let alertLevel = 0  // 0=安全, 1=正常, 2=警告, 3=危险
  let isSafe = true
  
  // ========== 1. 心率检测 ==========
  if (data.heartRate && data.heartRate.last > 0) {
    const hr = data.heartRate.last
    
    if (hr > 140) {
      // 危险：心率极高
      alerts.push({
        type: 'heart_rate_danger',
        level: 3,
        message: `⚠️ 心率极高：${hr} bpm（危险！立即停工）`,
        value: hr,
        threshold: 140
      })
      alertLevel = 3
      isSafe = false
    }
    else if (hr > 120) {
      // 警告：心率偏高
      alerts.push({
        type: 'heart_rate_high',
        level: 2,
        message: `⚠️ 心率偏高：${hr} bpm（需要休息）`,
        value: hr,
        threshold: 120
      })
      alertLevel = Math.max(alertLevel, 2)
      isSafe = false
    }
    else if (hr > 110) {
      // 注意：心率略高
      alerts.push({
        type: 'heart_rate_warning',
        level: 1,
        message: `💛 心率略高：${hr} bpm（注意观察）`,
        value: hr,
        threshold: 110
      })
      alertLevel = Math.max(alertLevel, 1)
    }
    else if (hr < 50) {
      // 警告：心率过低
      alerts.push({
        type: 'heart_rate_low',
        level: 2,
        message: `⚠️ 心率过低：${hr} bpm（可能异常）`,
        value: hr,
        threshold: 50
      })
      alertLevel = Math.max(alertLevel, 2)
      isSafe = false
    }
  }
  
  // ========== 2. 血氧检测 ==========
  if (data.bloodOxygen && data.bloodOxygen.value > 0) {
    const spo2 = data.bloodOxygen.value
    
    if (spo2 < 85) {
      // 危险：血氧极低
      alerts.push({
        type: 'blood_oxygen_danger',
        level: 3,
        message: `🚨 血氧极低：${spo2}%（危险！立即撤离）`,
        value: spo2,
        threshold: 85
      })
      alertLevel = 3
      isSafe = false
    }
    else if (spo2 < 90) {
      // 警告：血氧偏低
      alerts.push({
        type: 'blood_oxygen_low',
        level: 2,
        message: `⚠️ 血氧偏低：${spo2}%（需要补氧）`,
        value: spo2,
        threshold: 90
      })
      alertLevel = Math.max(alertLevel, 2)
      isSafe = false
    }
    else if (spo2 < 92) {
      // 注意：血氧略低
      alerts.push({
        type: 'blood_oxygen_warning',
        level: 1,
        message: `💛 血氧略低：${spo2}%（注意监测）`,
        value: spo2,
        threshold: 92
      })
      alertLevel = Math.max(alertLevel, 1)
    }
  }
  
  // ========== 3. 压力检测 ==========
  if (data.stress && data.stress.value > 0) {
    const stress = data.stress.value
    
    if (stress > 90) {
      // 危险：压力过大
      alerts.push({
        type: 'stress_danger',
        level: 3,
        message: `🚨 压力过大：${stress}（精神高度紧张）`,
        value: stress,
        threshold: 90
      })
      alertLevel = Math.max(alertLevel, 3)
      isSafe = false
    }
    else if (stress > 80) {
      // 警告：压力较大
      alerts.push({
        type: 'stress_high',
        level: 2,
        message: `⚠️ 压力较大：${stress}（需要缓解）`,
        value: stress,
        threshold: 80
      })
      alertLevel = Math.max(alertLevel, 2)
      isSafe = false
    }
    else if (stress > 70) {
      // 注意：压力偏高
      alerts.push({
        type: 'stress_warning',
        level: 1,
        message: `💛 压力偏高：${stress}（注意调节）`,
        value: stress,
        threshold: 70
      })
      alertLevel = Math.max(alertLevel, 1)
    }
  }
  
  // ========== 4. 深度检测（海拔为负） ==========
  if (data.barometer && data.barometer.altitude < 0) {
    const depth = Math.abs(data.barometer.altitude)
    
    if (data.barometer.altitude < -1000) {
      // 危险：深度过大
      alerts.push({
        type: 'depth_danger',
        level: 3,
        message: `🚨 深度过大：${depth}米（超限！）`,
        value: depth,
        threshold: 1000
      })
      alertLevel = Math.max(alertLevel, 3)
      isSafe = false
    }
    else if (data.barometer.altitude < -500) {
      // 警告：深度较大
      alerts.push({
        type: 'depth_warning',
        level: 2,
        message: `⚠️ 深度较大：${depth}米（接近限制）`,
        value: depth,
        threshold: 500
      })
      alertLevel = Math.max(alertLevel, 2)
      isSafe = false
    }
  }
  
  // ========== 5. 体温检测 ==========
  if (data.bodyTemperature && data.bodyTemperature.current > 0) {
    const temp = data.bodyTemperature.current
    
    if (temp > 38.5) {
      // 危险：体温过高
      alerts.push({
        type: 'temperature_high',
        level: 3,
        message: `🚨 体温过高：${temp}°C（可能发热）`,
        value: temp,
        threshold: 38.5
      })
      alertLevel = Math.max(alertLevel, 3)
      isSafe = false
    }
    else if (temp > 37.8) {
      // 警告：体温偏高
      alerts.push({
        type: 'temperature_warning',
        level: 2,
        message: `⚠️ 体温偏高：${temp}°C（注意观察）`,
        value: temp,
        threshold: 37.8
      })
      alertLevel = Math.max(alertLevel, 2)
      isSafe = false
    }
    else if (temp < 36.0) {
      // 警告：体温过低
      alerts.push({
        type: 'temperature_low',
        level: 2,
        message: `⚠️ 体温过低：${temp}°C（可能失温）`,
        value: temp,
        threshold: 36.0
      })
      alertLevel = Math.max(alertLevel, 2)
      isSafe = false
    }
  }
  
  // ========== 返回检测结果 ==========
  return {
    isSafe,                    // 是否安全
    alertLevel,                // 报警级别（0-3）
    alertMessage: alerts.map(a => a.message).join('; '),
    alerts,                    // 所有报警详情
    thresholds: SAFETY_THRESHOLDS  // 阈值配置
  }
}
```

### 阶段6: WebSocket实时推送

**执行文件**: `health-monitor-web/server/server.js`

**推送时机**:
- 收到新的手表数据后
- 报警状态变化时
- 设备上线/离线时

**详细流程**:

```javascript
// ========== 创建WebSocket服务器 ==========
const WebSocket = require('ws')
const wss = new WebSocket.Server({ server })

// 存储所有连接的客户端
const wsClients = new Set()

// ========== 处理新连接 ==========
wss.on('connection', (ws) => {
  console.log('🔌 新的WebSocket客户端连接')
  
  // 添加到客户端集合
  wsClients.add(ws)
  
  // 处理断开连接
  ws.on('close', () => {
    console.log('🔌 WebSocket客户端断开')
    wsClients.delete(ws)
  })
  
  // 处理错误
  ws.on('error', (error) => {
    console.error('WebSocket错误:', error)
    wsClients.delete(ws)
  })
})

// ========== 广播函数 ==========
function broadcastUpdate(data) {
  const message = JSON.stringify(data)
  
  console.log(`📡 广播更新给 ${wsClients.size} 个客户端`)
  
  // 遍历所有客户端
  wsClients.forEach(client => {
    // 只向已连接的客户端发送
    if (client.readyState === WebSocket.OPEN) {
      try {
        client.send(message)
      } catch (error) {
        console.error('发送失败:', error)
        wsClients.delete(client)
      }
    } else {
      // 清理未连接的客户端
      wsClients.delete(client)
    }
  })
}
```

## 🔄 数据更新机制

### 双重更新机制

系统采用**HTTP轮询 + WebSocket推送**双重更新机制，确保数据实时性和可靠性。

#### 1. HTTP定时轮询（基础）

```javascript
// 前端 Dashboard.vue

// 每5秒自动刷新一次
setInterval(() => {
  fetch('/api/realtime/all')
    .then(response => response.json())
    .then(data => {
      workers.value = data
    })
}, 5000)
```

**优点**:
- 简单可靠
- 不依赖WebSocket
- 确保数据最终一致性

**缺点**:
- 有延迟（最多5秒）
- 增加服务器负载

#### 2. WebSocket实时推送（增强）

```javascript
// 后端 server.js

// 收到手表数据后立即推送
broadcastUpdate({
  type: 'realtime_update',
  worker_id: 'W001',
  data: realtimeData
})

// 前端 Dashboard.vue

ws.onmessage = (event) => {
  const message = JSON.parse(event.data)
  updateWorkerData(message.worker_id, message.data)
}
```

**优点**:
- 零延迟（毫秒级）
- 节省带宽
- 实时性强

**缺点**:
- 需要保持连接
- 可能断线（需重连机制）

#### 3. 组合使用

```
手表推送 → 服务器接收 → WebSocket立即推送 → 前端实时更新
                       ↓
                   保存数据库
                       ↑
                   HTTP轮询（兜底）
```

```
      • 超时 → 网络问题

4. 检查数据是否存在
   ├─ 工人是否有实时数据？
   ├─ 查询: curl http://localhost:3000/api/realtime/W001
   └─ 如果没有数据，AI无法分析

5. 测试降级功能
   ├─ 即使AI失败，系统应该返回错误信息
   └─ 不会导致整个系统崩溃
```

## 📚 总结

### 系统核心流程

1. **数据采集**（手表）→ 每5分钟自动读取传感器
2. **数据传输**（BLE + HTTP）→ 手表→手机→服务器
3. **数据处理**（服务器）→ 安全检测 + 数据库存储
4. **实时推送**（WebSocket）→ 服务器→前端
5. **数据展示**（前端）→ 实时卡片 + 图表 + 报警
6. **AI分析**（DeepSeek）→ 深度分析 + 安全建议

### 数据流向

```
传感器 → 手表 → 手机 → 服务器 → 数据库
                              ↓
                         安全检测
                              ↓
                         WebSocket
                              ↓
                          前端展示
```

4. **WebSocket**: 双向实时通信
5. **Vue 3**: 现代前端框架
6. **DeepSeek AI**: 智能分析
