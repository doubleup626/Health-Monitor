# 项目总结

## 📊 项目概况

**项目名称：** ZeppOS 健康监测应用  
**版本：** v1.0.0  
**目标设备：** Active 2 (Round) 及其他支持 API_LEVEL 4.0+ 的 ZeppOS 设备  
**开发周期：** 完成  
**技术栈：** JavaScript, ZeppOS SDK, Node.js, Express, SQLite

## 🎯 项目目标

开发一款 ZeppOS 应用，实现以下功能：

1. ✅ 采集人体健康数据：心率、血氧、压力、体温、步数
2. ✅ 采集环境数据：气压、海拔高度
3. ✅ 后台持续运行（息屏也可采集）
4. ✅ 定时采集数据（默认 5 分钟）
5. ✅ 定时上传数据到服务器（默认 15 分钟）
6. ✅ 本地数据缓存（防止丢失）
7. ✅ 友好的用户界面

## 🔑 核心技术点

### 1. App Service（后台服务）

**技术难点：** 息屏后持续运行

**解决方案：**
- 使用 ZeppOS 3.0+ 引入的 App Service 功能
- 配置 `device:os.bg_service` 权限
- 用户授权后可在后台持续运行

**实现代码：**
```javascript
AppService({
  onInit(e) {
    // 创建系统定时器
    collectTimerId = createSysTimer(true, INTERVAL, collectData);
  }
});
```

### 2. 系统定时器（息屏可用）

**技术难点：** 普通定时器息屏后停止

**解决方案：**
- 使用 API 4.0 新增的 `createSysTimer`
- 系统级定时器，不受屏幕状态影响
- 支持周期性和一次性定时

**实现代码：**
```javascript
import { createSysTimer, stopSysTimer } from "@zos/timer";

const timerId = createSysTimer(
  true,              // 周期性
  5 * 60 * 1000,     // 5分钟
  () => {
    // 采集数据
  }
);
```

### 3. 传感器数据采集

**技术难点：** 多种传感器的统一管理

**解决方案：**
- 创建 `SensorCollector` 类封装所有传感器
- 统一的初始化、采集、销毁接口
- 错误处理和日志记录

**支持的传感器：**
- HeartRate（心率）
- BloodOxygen（血氧）
- Stress（压力）
- BodyTemperature（体温）
- Step（步数）
- Barometer（气压、海拔）

### 4. BLE 通信

**技术难点：** Device App 和 Side Service 通信

**解决方案：**
- Device App 使用 `@zos/ble` 模块发送数据
- Side Service 使用 `messaging.peerSocket` 接收数据
- 数据格式：JSON 字符串转 ArrayBuffer

**数据流：**
```
Device App → BLE → Zepp App → Side Service → HTTP → Server
```

### 5. 数据缓存机制

**技术难点：** 网络不可用时防止数据丢失

**解决方案：**
- 本地存储最多 100 条数据
- 上传成功后清空缓存
- 上传失败则保留本地数据

### 6. 权限管理

**关键权限：**
- `data:user.hd.*`: 健康数据读取
- `device:os.barometer`: 气压计
- `device:os.bg_service`: 后台服务（需用户授权）
- `device:os.local_storage`: 本地存储
