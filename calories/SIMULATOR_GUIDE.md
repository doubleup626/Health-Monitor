# 模拟器使用指南

## 🎯 模拟器的作用

ZeppOS 模拟器主要用于：
- ✅ **UI 界面开发** - 快速查看界面效果
- ✅ **交互逻辑测试** - 测试按钮、页面跳转
- ✅ **基础功能验证** - 验证应用能否正常启动
- ✅ **快速迭代** - 无需真机即可测试

## ⚠️ 模拟器的限制

### 完全支持的功能
- ✅ UI 界面渲染
- ✅ 文本显示
- ✅ 按钮点击
- ✅ 页面跳转
- ✅ 本地存储（LocalStorage）
- ✅ 基础传感器数据（返回模拟值）

### 部分支持的功能
- ⚠️ **App Service（后台服务）**
  - 可以启动和停止
  - **但定时器可能不工作**（这是当前的问题）
  - 可以手动触发采集
  
- ⚠️ **系统定时器（createSysTimer）**
  - 可能返回 null/undefined
  - 可能无法在后台正常工作
  - 这是模拟器的技术限制

- ⚠️ **传感器数据**
  - 返回模拟值（不是真实数据）
  - 部分传感器可能返回 0 或 undefined

### 不支持的功能
- ❌ BLE 蓝牙通信（受限）
- ❌ 网络请求到外部服务器
- ❌ GPS 定位
- ❌ 真实的息屏后台运行

## 📱 当前应用在模拟器中的表现

### ✅ 可以正常使用的功能

#### 1. 手动采集数据
**操作：** 点击"手动采集一次"按钮

**效果：**
```
✅ 采集成功
✅ 数据显示更新
✅ 本地存储保存
✅ "已采集数据"数量增加
```

**日志示例：**
```
[sensors] [-] 传感器初始化成功
[sensors] [-] 心率 - 当前: 75, 最近: 75
[sensors] [-] 步数 - 当前: 300, 目标: 3000
[sensors] [-] 数据采集完成
```

#### 2. 查看历史数据
**操作：** 打开应用

**效果：**
- 显示上次采集的数据
- 显示已采集的数据条数
- 这是正常行为（不是 bug）

#### 3. 数据持久化
**特点：**
- 关闭应用后数据不丢失
- 重新打开显示历史数据
- 数据保存在 `health_data.txt`

### ⚠️ 部分工作的功能

#### 后台服务启动
**操作：** 点击"开始采集"

**当前状态：**
```
✅ 服务可以启动
✅ 首次采集成功
⚠️ 定时器可能创建失败
❌ 自动定时采集可能不工作
```

**日志分析：**
```
✅ [data-collector.service] [-] Data Collector Service onInit
✅ [sensors] [-] 传感器初始化成功
⚠️ [定时器创建返回 null/undefined，可能是模拟器限制]
✅ [Data Collector Service 启动成功]
```

**含义：**
- 服务启动成功（显示"result":true）
- 但定时器无法创建
- 所以只会采集一次，后续不会自动采集

### ❌ 不工作的功能

#### 自动定时采集
**预期：** 每 5 分钟自动采集一次

**实际：** 在模拟器中不会自动采集

**原因：** `createSysTimer` 在模拟器中可能无法正常工作

## 🎓 如何在模拟器中测试应用

### 推荐的测试流程

#### 第 1 步：测试 UI 界面
```
✅ 应用可以启动
✅ 界面显示正常
✅ 按钮可以点击
✅ 文字显示清晰
```

#### 第 2 步：测试手动采集
```bash
操作步骤：
1. 点击"手动采集一次"
2. 查看日志是否有采集成功的信息
3. 查看界面是否显示数据
4. 多次点击，确认数据数量增加
```

**预期结果：**
- 每次点击都能采集成功
- 界面显示最新数据
- "已采集数据"数量递增

#### 第 3 步：测试数据持久化
```bash
操作步骤：
1. 手动采集几次数据
2. 关闭应用
3. 重新打开应用
4. 查看数据是否还在
```

**预期结果：**
- 数据没有丢失
- 显示上次的数据
- 数量正确

#### 第 4 步：测试服务启停
```bash
操作步骤：
1. 点击"开始采集"（会看到定时器警告）
2. 点击"停止采集"
3. 反复测试启停
```

**预期结果：**
- 服务可以启动和停止
- 状态显示正确
- 没有崩溃

## 💡 模拟器测试建议

### 1. 专注于可测试的功能

**应该测试：**
- ✅ UI 布局是否美观
- ✅ 按钮是否响应
- ✅ 数据显示是否正确
- ✅ 手动采集是否工作
- ✅ 本地存储是否正常

**暂时忽略：**
- ⏸️ 自动定时采集（真机测试）
- ⏸️ 后台运行（真机测试）
- ⏸️ 数据上传（真机测试）

### 2. 理解错误日志

**正常的警告（可以忽略）：**
```
⚠️ 定时器创建返回 null/undefined，可能是模拟器限制
⚠️ 将使用手动采集模式（模拟器限制）
```

**需要关注的错误：**
```
❌ 传感器初始化失败
❌ 数据采集失败
❌ 界面显示异常
```

### 3. 调整配置以适应模拟器

**可选：缩短采集间隔（方便测试）**

编辑 `app-service/data-collector.js`：
```javascript
// 改为 30 秒，方便真机测试
const COLLECT_INTERVAL = 30 * 1000; // 30秒
```

**注意：** 记得测试完改回 5 分钟！
