# 快速入门指南

本指南将帮助你在 10 分钟内运行起健康监测应用。

## 📋 前置检查清单

在开始之前，请确保：

- ✅ 已安装 Zeus CLI 开发工具
- ✅ 有一台支持 ZeppOS 4.0+ 的手表（如 Active 2）
- ✅ 电脑已安装 Node.js v14+
- ✅ 手机已安装 Zepp App

## 🎯 步骤 1: 启动服务器（3 分钟）

### 1.1 进入服务器目录

```bash
cd calories/server-example
```

### 1.2 安装依赖

```bash
npm install
```

这会安装：
- `express`: Web 服务器框架
- `body-parser`: 请求体解析
- `sqlite3`: 数据库

### 1.3 启动服务器

```bash
npm start
```

你会看到类似输出：

```
==================================================
健康监测数据服务器已启动
监听地址: http://0.0.0.0:3000
API 端点:
  - GET  /api/ping         : 测试连接
  - POST /api/health-data  : 接收健康数据
  - GET  /api/health-data  : 查询健康记录
  - GET  /api/stats        : 获取统计信息
==================================================

本机 IP 地址:
  WLAN: http://192.168.1.100:3000
  
请在 app-side/index.js 中配置正确的服务器地址
==================================================
```

**⚠️ 重要：记下你的 IP 地址**（如上例中的 `192.168.1.100`）

### 1.4 测试服务器

打开浏览器，访问：

```
http://你的IP:3000/api/ping
```

如果看到：
```json
{"success":true,"message":"Server is running","timestamp":1234567890}
```

说明服务器运行正常！✅

## 🎯 步骤 2: 配置应用（2 分钟）

### 2.1 打开配置文件

用编辑器打开：

```
calories/app-side/index.js
```

### 2.2 修改服务器地址

找到第 4 行：

```javascript
const SERVER_URL = "http://192.168.1.100:3000/api/health-data";
```

**将 `192.168.1.100` 改为你在步骤 1.3 中看到的 IP 地址。**

### 2.3 保存文件

按 `Ctrl+S` (Windows) 或 `Cmd+S` (Mac) 保存。

## 🎯 步骤 3: 编译应用（2 分钟）

### 3.1 返回项目根目录

```bash
cd ..  # 从 server-example 返回到 calories
```

### 3.2 启动开发模式

```bash
zeus dev
```

这会：
1. 编译应用代码
2. 启动开发服务器
3. 等待手表连接

你会看到：

```
[zeus] Building...
[zeus] Build success!
[zeus] Server running at http://localhost:8081
[zeus] Waiting for device...
```

## 🎯 步骤 4: 安装到手表（3 分钟）

### 4.1 连接手表到开发服务器

**方法 1：通过模拟器（推荐新手）**

1. 启动 ZeppOS 模拟器
2. 在模拟器中输入开发服务器地址
3. 应用会自动安装

**方法 2：通过真机（推荐调试）**

1. 手表连接到电脑的 Wi-Fi（确保在同一网络）
2. 在手表上打开"开发者模式"
3. 在手表中输入开发服务器地址 `http://你的电脑IP:8081`
4. 应用会自动安装

### 4.2 打开应用

在手表上找到"健康监测"应用并打开。

## 🎯 步骤 5: 使用应用（30 秒）

### 5.1 授予权限

首次启动时，点击"开始采集"会提示授予权限。

**选择"允许"** 以授予后台服务权限。

### 5.2 开始采集

权限授予后，服务会自动启动。你会看到：

```
● 数据采集中
```

### 5.3 查看数据

等待几秒钟，界面会显示采集到的数据：

```

最后采集时间:
2024-10-07 14:30:00

❤️ 心率: 75 bpm

👣 步数: 5000
```

### 5.4 手动采集（可选）

点击"手动采集一次"按钮，可以立即采集一次数据。

## 🎯 步骤 6: 验证数据上传（1 分钟）

### 6.1 检查服务器日志

回到运行服务器的终端窗口，你应该能看到：

```
[POST] 收到健康数据
请求体: { ... }
开始保存 1 条数据
数据保存完成: 成功 1 条, 失败 0 条
```

### 6.2 查询数据

在浏览器中访问：

```
http://你的IP:3000/api/health-data?limit=5
```

你会看到保存的数据：

```json
  "data": [

      "device_id": "device123",
      "timestamp": 1234567890000,
```

## 🐛 常见问题

### Q: 服务器启动失败，提示端口被占用

**A:** 修改 `server-example/server.js` 中的端口：

```javascript
const PORT = 3001; // 改为其他端口
```

然后记得在 `app-side/index.js` 中也改为对应端口。

### Q: 手表连接不上开发服务器

**A:** 检查：
1. 手表和电脑在同一 Wi-Fi 网络
2. 防火墙没有阻止 8081 端口
3. IP 地址输入正确

### Q: 传感器数据显示 0

**A:** 
1. 确保手表正确佩戴
2. 等待 1-2 分钟让传感器初始化
3. 尝试手动采集一次

### Q: 数据无法上传到服务器

**A:** 检查：
1. `app-side/index.js` 中的服务器地址是否正确
2. 服务器是否正在运行
3. 手机和服务器在同一网络（注意：手表通过手机上传数据）
4. 在浏览器测试 `/api/ping` 接口
